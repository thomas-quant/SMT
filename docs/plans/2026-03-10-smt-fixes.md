# SMT Tracking And API Fixes Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix the five review findings by rewriting break tracking to be asset-aware, tightening detector validation, protecting registry state, correcting FVG timestamps, and updating the public API/docs.

**Architecture:** The manager will derive current-candle data from both asset DataFrames and push asset-aware invalidation checks into `SMTBreak`. Detector signals will explicitly declare the invalidation asset and break direction. Registry reads will return deep copies to preserve state ownership, and validation will raise `ValueError` for malformed input instead of failing late.

**Tech Stack:** Python 3.9+, pandas, numpy, pytest
